### 2. 예측 모델이 레포에 없음 → 경로에 혼잡도 못 붙임

| | |
|--|--|
| **문제** | main(및 다른 브랜치)에 AI팀 LightGBM 산출물이 없어, 경로 검색 결과가 ODsay 시간/환승 위주이고 **모델 혼잡 응답과 UI가 안 맞음** |

| **해결** | `feature/integrate-congestion-model`에서 `backend/models/*` + `congestion_model.py`/`predict.py` 연동. ODsay 경로에 **배치/시간대 혼잡 API** 붙임. 프론트 `apply-congestion`·차트·카드·타임라인 정리. 칸별 히트맵은 범위 밖(열차별은 가능하면 유지). **[PR #2](https://github.com/davidmh0203/SUBWAY-PREDICT/pull/2)** 머지·Deploy. 스모크: mock / 최소 live(`smoke-*-predict`) |

### 3. ODsay 호출 한도(과호출) 위험

| | |
|--|--|
| **문제** | 경로·혼잡 UX를 손보면 슬라이더/재검색마다 ODsay를 다시 치면 **일일 한도**에 막힐 수 있음 |
| **원인** | 경로 탐색과 혼잡 갱신을 같은 “재요청”으로 묶으면 호출이 폭발 |
| **해결** | 경로(ODsay)는 **검색 시 최소 호출**. 시간 슬라이더는 **이미 받은 경로·모델 혼잡만 다시 맞춤**(ODsay 재호출 없음). 안내 문구 “슬라이더는 차트 강조만…”은 UX상 오해 유발 → 제거. live 테스트도 횟수 제한. 키 없을 때는 픽스처/목업으로 검증 |

---

### 4. 시간 슬라이더 ≠ 막대/차트 체감

| | |
|--|--|
| **문제** | 슬라이더를 움직여도 막대·그래프가 안 바뀌면 “왜 있지?” / 반대로 카드와 붙어 있어 **막대까지 바뀔 것 같음** |
| **원인** | ODsay 재호출을 끊으면 경로 geometry는 고정. 혼잡·차트만 모델 시간대로 바꿀 수 있음. UI 카피가 역할을 잘못 설명하기도 함 |
| **해결** | 슬라이더 → **경로 카드 혼잡 재적용**(모델/배치). 차트는 선택 시각 강조·구간이 사용자 설정 시각 전후로 맞게 조정하는 방향으로 진행. 출/퇴근 라벨 오표기(심야를 퇴근으로 표기)는 **출·퇴근 시간대 기준**으로 재정의 요청(로컬/`TimeBottomSheet` 수정 중) |

---

### 5. 추천 경로 카드·상세 도식 UX

| | |
|--|--|
| **문제** | 타임라인 막대에 아이콘 원 위치가 애매하고, 상세 경로 중간 역 노드가 선 중심에서 離脱. 글씨 여백·원/막대 비율도 답답 |
| **원인** | 막대·원 CSS/레이아웃이 네이버식 비율과 다름. 도식 노드 정렬이 라인 중심이 아님 |
| **해결** | 원을 막대 **시작 끝에 밀착**, 원 크기 ~1.2배·막대 두께·여백 조정(네이버 레퍼런스). 중간 노드를 **호선 라인 중심**에. 역별 혼잡도 토글을 경로 다이어그램 토글과 **분리**해 각각 접기/펼치기 |

---

### 6. 홈 “정체 예보” 카드가 목업뿐

| | |
|--|--|
| **문제** | 메인 이벤트/예보 카드가 목업만 보여 **공휴일·날씨·돌발·시위/행사**가 안 갱신됨 |
| **원인** | 카드 데이터가 정적 mock. SPATIC/NTCE CSV는 전날 수집돼 있어도 홈 API에 안 묶임. 전부 DB화할지는 미정 |
| **해결** | (진행 중·로컬) 백엔드 `forecast_service`가 **공휴일 캘린더 + 날씨 API + ntce(+CSV) + SPATIC CSV**를 위치·시간 기준으로 합성 → `/forecast`류 라우터 + 프론트 `forecast-client`. ODsay는 건드리지 않음. CSV는 서버에 파일로 로드(`csv_store`). **아직 main 미커밋** |

---

### 7. SPATIC만으로는 과거 집회·신고인원 부족

| | |
|--|--|
| **문제** | SPATIC 보드는 ~2024-03부터라 과거가 비고, 기존 CSV에 **신고인원** 없음. 서울경찰청(SMPA)에는 2011~ 자료가 있으나 **HWP/PDF/JPG 첨부** |
| **원인** | 사이트별 보관 형태 다름. 첨부는 텍스트 스크랩만으로 인원·일정을 못 뽑음 |
| **해결** | SMPA 목록→첨부 다운로드→HWP/PDF 파싱 파이프라인 (`crawl/parse-smpa-assem`). SPATIC과 병합해 `assem-events.csv` 확장. 신고인원 필드 전 행 반영 시도. **스냅샷(로컬):** events ~46,466 / SMPA assemblies ~42,075 / 인원 확보 ~27,297 (`smpa-parse-report.json`) |
