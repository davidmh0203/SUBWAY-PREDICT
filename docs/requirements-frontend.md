## 1. 개요

수도권 전철 혼잡도 예측 목업의 **프론트엔드 UI/UX 개선** 요구사항이다.  
로그인·회원 기능은 이번 범위에 포함하지 않으며, 데이터 제약으로 **노선도는 서울시(서울메트로) 관할 역만** 다룬다.

### 관련 화면 (현재 코드 기준)

| 화면명 | 라우트/뷰 | 주요 컴포넌트 |
|--------|-----------|---------------|
| 홈 | `#home` | `HomeScreen.jsx` |
| 경로 결과 | `#results` | `RouteResultsScreen.jsx` |
| 경로 상세 | `#detail` | `RouteDetailScreen.jsx`, `RouteSchematic.jsx`, `CarCongestionHeatmap.jsx` |
| 노선도 | `#macro` | `MacroViewScreen.jsx`, `InteractiveMetroMap.jsx` |

## 2. 이번 범위 (Must)

### 2.1 전체 — 반응형

**요구사항**

- PC(데스크톱)와 모바일(앱/웹)에서 **화면 크기에 맞게 레이아웃이 달라지도록** 반응형을 적용한다.
- 현재 `App.jsx`는 `max-w-md` 중심의 모바일 폭 고정에 가깝다. 뷰포트가 넓어질 때 콘텐츠 폭·여백·배치가 자연스럽게 확장되도록 한다.

**완료 기준 (초안)**

- 모바일(~390px): 기존 목업 UX 유지
- 태블릿·PC(768px+): 레이아웃이 깨지지 않고 가독성 유지
- 하단 네비게이션·카드·차트·지도 영역이 뷰포트에 맞게 조정됨

**확인 필요**

- PC에서도 하단 탭 네비를 유지할지, 사이드바 등으로 바꿀지
- 최대 콘텐츠 폭 상한(예: `max-w-lg`, `max-w-2xl`) 지정 여부

---

### 2.2 홈 (`HomeScreen`)

#### 2.2.1 회원 아이콘 제거

**요구사항**

- 헤더 우측 **프로필(회원) 아이콘**을 UI에서 제거한다.
- 로그인·회원가입 기능은 구현하지 않는다.

**현재 상태**

- `HomeScreen.jsx` 헤더: 설정(좌) · 타이틀 · User 아이콘(우)

**완료 기준**

- 홈 화면에 회원/프로필 진입 UI가 보이지 않음

---

#### 2.2.2 자주 가는 경로 섹션 제거

**요구사항**

- **「자주 가는 쾌적 경로」** 섹션 전체를 이번 버전에서 제거한다.
- 로그인 없이 즐겨찾기를 쓸 수 없으므로 당분간 미노출.

**현재 상태**

- `HomeScreen.jsx` 하단: Star 아이콘 + 샘플 카드(집 → 회사)

**완료 기준**

- 홈에 자주 가는 경로 UI가 없음

---

#### 2.2.3 오늘의 정체 예보 — 가로 스와이프 캐러셀

**요구사항**

- 상단 **「오늘의 정체 예보」** 카드 영역:
  - 이벤트가 많아져도 **세로로 길어지지 않음** (카드 높이·영역 고정 또는 일정)
  - 모바일에서 **좌우 스와이프(캐러셀)** 로 다른 이벤트 확인
  - **첫 카드**에는 **큰 이벤트(중요 이벤트)** 위주 노출
  - 추가 이벤트가 있음을 알 수 있는 **힌트** 제공  
    (예: 페이지 인디케이터(점), 다음 카드 일부 노출(peek), 화살표 등)

**현재 상태**

- 단일 `Card`에 텍스트 2줄 고정 표시 (비·콘서트 예시)

**완료 기준**

- 이벤트 2개 이상일 때 세로 스크롤 없이 가로로 넘겨 볼 수 있음
- 첫 슬라이드가 주요 이벤트를 반영
- 더 볼 내용이 있음을 사용자가 인지할 수 있는 시각적 힌트 존재

**확인 필요**

- 「큰 이벤트」정렬 기준: 혼잡도 예측 %, 인원 규모, 수동 `priority` 필드 등
- 이벤트 데이터 소스: mock 배열로 우선 구현할지

### 2.4 경로 상세 (`RouteDetailScreen`)

#### 2.4.1 칸별 혼잡도 제거 → 열차 혼잡도만 표시

**요구사항**

- **역×칸(1~8호차) 히트맵** 형태의 칸별 혼잡도 UI를 제거한다.
- **열차(구간/역) 단위 혼잡도**만 표시한다.
- 혼잡 **정도 표현 방식**(색상 단계, `CrowdBlock` 스타일, `CROWD_COLORS` 등)은 **칸별 UI에서 쓰던 방식을 그대로 재사용**한다.

**현재 상태**

- `CarCongestionHeatmap.jsx`: 역 목록 + 8칸 그리드
- `CongestionLegend.jsx`: 색상 바 + (비 compact 시) 아이콘 라벨

**완료 기준**

- 칸 번호(1~8) 열이 보이지 않음
- 역(또는 구간)마다 열차 혼잡도가 색상 블록 등 기존 스타일로 표시됨

**확인 필요**

- 「열차 혼잡도」단위: **역별 1값**인지, **구간별 1값**인지
- mock 데이터(`crowd-data.js`) 구조 변경 필요 여부

---

#### 2.4.2 혼잡도 레전드 — 텍스트 + %

**요구사항**

- 혼잡도 레전드에 **텍스트 라벨과 %** 를 함께 표시한다.
- 예: 여유 / 보통 / 혼잡 / 매우혼잡 + 해당 구간 % 범위 또는 대표 수치

**현재 상태**

- `CongestionLegend`: compact 모드는 색상 바만, full 모드는 아이콘+짧은 라벨 (`CROWD_LABELS`)

**완료 기준**

- 상세 화면 혼잡도 영역에서 레전드를 보면 **이름 + %** 정보를 확인할 수 있음

**확인 필요**

- % 범위 정의 (예: 0–80% 여유, 80–100% 보통 …) — `congestion.js` 기존 정의와 통일할지

---

#### 2.4.3 경로 상세 세로 길이 축소 — 접기/토글 UX

**요구사항**

- 경로 상세가 **아래로 너무 길어지는 문제** 해결.
- 방안 (택1 또는 조합, 플랜 단계에서 결정):
  - **A.** 경로를 **가로로 길게** 배치 (가로 스크롤/타임라인)
  - **B.** **네이버 지도 앱 스타일 토글**: 출발지·도착지·환승역만 크게 표시, 중간 역은 접힌 상태 → 토글 시 펼쳐서 표시

**현재 상태**

- `RouteSchematic.jsx`: 세로 다이어그램
- `CarCongestionHeatmap`: 역마다 행 추가 → 역 수가 많을수록 세로로 길어짐

**완료 기준**

- 기본 화면에서 스크롤 부담이 줄어듦
- 출발·도착·환승역이 한눈에 들어옴
- 필요 시 중간 역 상세를 볼 수 있음

**확인 필요**

- A/B 중 우선안
- `RouteSchematic`과 혼잡도 영역을 **같은 토글 패턴**으로 묶을지, 각각 적용할지

---

#### 2.4.4 경로 상세 하단 — 경로 전용 미니 지도

**요구사항**

- 경로 상세 **하단에 지도** 추가.
- **전체 노선도 탭(`#macro`)으로 이동하지 않아도** 해당 화면에서 지도 확인 가능.
- 지도에는 **현재 경로에 해당하는 노선·역만** 강조/표시 (전체 수도권 맵이 아닌 경로 맥락).

**현재 상태**

- 노선도는 `MacroViewScreen` + `InteractiveMetroMap` 에만 존재
- 상세 화면에는 다이어그램 + 히트맵만 있음

**완료 기준**

- 상세 화면 하단에 지도 영역 존재
- 선택한 경로의 역·호선이 지도에 반영됨
- 노선도 탭 없이도 경로 공간 관계 파악 가능

**확인 필요**

- `InteractiveMetroMap` 재사용 vs 경로 전용 축소 컴포넌트 신규
- 지도 높이·접기 가능 여부

---

### 2.5 노선도 (`MacroViewScreen`)

#### 2.5.1 서울시(서울메트로) 역만 표시

**요구사항**

- 데이터 부족으로 **서울시에서 관리하는 역 정보만** 노선도에 노출한다.
- 서울메트로 관련 데이터셋에 없는 역(타 광역·경기 등)은 이번 목업에서 제외하거나 비표시.

**현재 상태**

- `metro-network.js`, `InteractiveMetroMap.jsx` 등에 수도권 전체 망 데이터 가능성 있음

**완료 기준**

- 노선도에 서울시 관할이 아닌 역이 노출되지 않거나, 데이터 범위가 문서·UI에 명시됨
- 경로 검색/표시와 데이터 범위가 모순되지 않음 (검색 가능 역 vs 지도 역)

**확인 필요**

- 「서울시만」의 정확한 역 목록 기준 (CSV/API/mock 화이트리스트)
- 경기·인천 역이 경로에 포함될 때 지도/상세에서 어떻게 처리할지

## 5. 확인이 필요한 질문 (플랜 단계용)

1. **반응형**: PC 레이아웃은 모바일 확대판인가, 2열·사이드 패널 등 별도 구성인가?
2. **정체 예보**: 「큰 이벤트」우선순위 규칙은?
3. **열차 혼잡도**: 역당 1값인가, 호선 구간당 1값인가?
4. **레전드 %**: 고정 구간(여유 0–80%)인가, 실측 예측값 표시인가?
5. **경로 접기**: 네이버 지도식 토글(B)을 1순위로 할지, 가로 타임라인(A)을 병행할지?
6. **상세 지도**: 기존 `InteractiveMetroMap` 임베드 vs 경로 하이라이트 전용 경량 지도?
7. **서울시 역 범위**: 공식 역 ID 목록 소스는 mock에 하드코딩할지, 기존 `metro-network` 필터링할지?
8. **경로에 비서울 역 포함 시**: 검색은 허용하되 지도만 생략할지, 경로 자체를 서울 역만으로 제한할지?

## 6. 수용 기준 요약 (Acceptance Criteria)

| # | 항목 | 수용 기준 |
|---|------|-----------|
| AC-01 | 반응형 | 모바일·PC에서 레이아웃 깨짐 없음 |
| AC-02 | 홈 헤더 | 회원 아이콘 없음 |
| AC-03 | 홈 즐겨찾기 | 자주 가는 경로 섹션 없음 |
| AC-04 | 정체 예보 | 가로 스와이프, 고정 높이, 주요 이벤트 우선, 더보기 힌트 |
| AC-05 | 상세 혼잡도 | 칸별 제거, 열차 단위만, 기존 색상 표현 유지 |
| AC-06 | 레전드 | 텍스트 + % 표시 |
| AC-07 | 상세 길이 | 접기/토글 또는 가로 배치로 세로 스크롤 완화 |
| AC-08 | 상세 지도 | 하단 경로 전용 지도, 노선도 탭 불필요 |
| AC-09 | 노선도 | 서울시 관할 역만 표시 |
| AC-10 | 빌드 | `npm run build` 성공 |

## 7. 원문 대조

<details>
<summary>사용자 원문 메모</summary>

### 프론트 수정사항

전체: 반응형으로 → pc에서, 앱에서 크기 달라지게.

#### 홈

- 회원 아이콘 빼기 , 자주가는 경로 빼기 → 아직 로그인/회원가입 기능 안하는걸로 → 시간 남으면 추가 (자주가는 경로)
- 상단의 **오늘의 정체 예보** 카드 이벤트 많아지면 텍스트와 카드가 아래로 길어지기 x, 대신 모바일에서 사용하기 편하기 우측으로 넘기면 다른 이벤트들 확인 할 수 있게. 처음 보이는 카드에는 큰 이벤트 위주로 보여주고 그외 넘어가면 우측으로 넘겨서 추가 이벤트들 볼 수 있게. 그리고 넘기면 이벤트 더 있다고 알수있게 힌트나 그런거 달아두기

#### 경로

(수정 항목 없음)

#### 상세

칸별 혼잡도 빼기, 열차혼잡도만 적용. 열차혼잡도에 칸별혼잡도에 사용하는 혼잡 정도 표현하는 방식은 그대로 가져가기. 혼잡도 정도 표현하는 레전드에 텍스트+ % 달기

경로 상세 아래로 너무 길어지니 해결방안→ 옆으로 길게 하거나, 네이버 지도 처럼 토글 ( 출발지, 도착지, 환승역만 크게 보게 하고 나머지 그 사이의 경로는 토글 눌렀을때 볼 수 있게)

경로 상세에 방금 말한 거 말고도 그 하단에 지도 추가 . 굳이 전체 노선도 페이지로 안가고 거기서 볼 수 있게 . 그러면 경로에 해당하는 노선만 볼 수 있게

#### 노선도

데이터가 부족해서 서울시것만 (서울메트로 관련 데이터에는 서울시에서 관리하는 역 정보만 나옴)
