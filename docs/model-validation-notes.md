# 혼잡도 모델 검증 메모 (2026-07-20)

발표·주간보고용 요약. 상세는 [`backend/models/README.md`](../backend/models/README.md) 참고.

## 한 줄 요약

모델이 **상대적 혼잡 순위(강남·연신내 높음)** 는 어느 정도 맞추지만, **절대 혼잡도 %·레벨의 정확도는 아직 검증이 부족**하다.

## 실측 UI에서 본 패턴 (18:30 출발)

- **높게 나온 구간:** 강남(84%) → 교대(88%) → 신사(95%) → 압구정(100%), 연신내(71%) 등
- **낮게 나온 구간:** 경복궁(33%), 안국(45%), 독립문(42%) 등

## 왜 애매한가

1. **월요일 + 장마·우중충 날씨** — 관광·외출 수요 감소가 반영됐는지, 아니면 모델이 과소 추정하는지 구분이 어렵다.
2. **역·호선 집계 방식** — 호선별이 아닌 **역 전체 합산**이면 관광역(경복궁)이 통근·환승 역과 섞여 %가 낮게 보일 수 있다.
3. **지표 정의** — 학습은 증감 분류, UI는 베이스라인 대비 %라 “체감 혼잡”과 수치가 어긋날 수 있다.

## 팀 공유 시 표현 예시

> “강남·연신내 등 혼잡 축은 방향성이 맞는 것 같고, 경복궁처럼 관광 비중이 큰 역은 요일·날씨·호선 집계 영향으로 해석이 어렵다. 정량 평가와 호선별 예측은 다음 스프린트 과제다.”
