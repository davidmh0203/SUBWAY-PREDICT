# 혼잡도 모델 검증 메모 (2026-07-20)

발표·주간보고용 요약. 상세는 [`backend/models/README.md`](../backend/models/README.md) 참고.

## 한 줄 요약

모델이 **상대적 혼잡 순위(강남·연신내 높음)** 는 어느 정도 맞추지만, **절대 혼잡도 %·레벨은 이벤트 데이터 오염** 때문에 신뢰하기 어렵다. **원인 파악 완료 — AI팀 담당자가 베이스라인·이벤트 보정 개선 예정.**

## 실측 UI에서 본 패턴 (18:30 출발)

- **높게 나온 구간:** 강남(84%) → 교대(88%) → 신사(95%) → 압구정(100%), 연신내(71%) 등
- **낮게 나온 구간:** 경복궁(33%), 안국(45%), 독립문(42%) 등

## 원인 (2026-07-20 업데이트)

**확인된 1차 원인 — 이벤트로 인한 베이스라인·평균 오염**

- 행사·공연·집회·축제 등 **외부 이벤트가 역별 평균·베이스라인을 끌어올리면**, 이벤트가 없는 평범한 날의 혼잡도 %가 **상대적으로 낮게** 나온다.
- 경복궁·안국처럼 관광 비중이 큰 역은 이벤트가 적은 날에도, **과거 이벤트가 섞인 평균**과 비교되면 지표가 왜곡될 수 있다.
- 반대로 강남·연신내 등은 통근·환승 축이라 이벤트 오염 영향이 상대적으로 덜해 **방향성은 맞는 것처럼** 보일 수 있다.

**보조 가설 (미확정)**

1. **월요일 + 장마·우중충 날씨** — 당일 수요 감소가 반영됐는지 추가 확인 필요.
2. **역·호선 집계** — 호선별이 아닌 역 전체 합산이면 % 해석이 더 어려울 수 있음.
3. **지표 정의** — 학습은 증감 분류, UI는 베이스라인 대비 %.

## 대응

- **AI팀 담당자가 이벤트 보정·베이스라인 정제를 개선**한다고 함 → 반영 전까지 현재 수치는 **참고용**으로만 사용.
- 프론트·백엔드는 기존 API·UI 유지, 모델 아티팩트 교체만 기다림.

## 팀 공유 시 표현 예시

> “혼잡도 %가 이상한 건 모델 자체보다 **이벤트가 평균을 올려서 베이스라인이 오염**된 영향이 크다. 강남·연신내는 방향은 맞는 편이고, 경복궁 등은 이벤트 보정 전까지 해석에 주의. 담당 팀원이 이벤트 처리 개선 중.”
