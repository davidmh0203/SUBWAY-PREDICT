## 문제 → 원인 → 해결

### 1. Vercel 자동 배포 / API 키

| | |
|--|--|
| **문제** | GitHub Actions로 Vercel 배포를 걸어 뒀는데, 로그상 토큰·인증 쪽이 불안하고 “키를 안 넣어서 깨진 건가?”로 보임 |
| **원인** | Deploy는 `VERCEL_TOKEN`(+ `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`)이 repo Secrets에 있어야 함. 과거(7/8~9) Deploy 실패 이력이 있고, 토큰 만료·스코프·미등록이 흔함. **7/13 당일 새 Deploy 런은 없음**(main push 없음) |
| **해결** | 토큰 재발급 절차·스코프(계정/프로젝트) 의미 정리 → GitHub Secrets에 `VERCEL_TOKEN` 등록. `ORG_ID`/`PROJECT_ID`는 프로젝트 불변이면 유지. passkey·recovery codes는 배포 키와 별개로 안내 |

### 2. Slider: 부모가 넘긴 props vs 자식 시그니처

| | |
|--|--|
| **문제** | 부모는 `min`/`max`/`step`/`value`/`onValueChange` 등 여러 개를 주는데, 자식은 `({ className, ...props }, ref)`처럼 보여 “왜 개수가 안 맞지?” |
| **원인** | React에서 나머지 props는 `...props`로 한 덩어리. `className`만 스타일용으로 따로 빼고, 나머지는 Radix/`SliderPrimitive`로 전달 |
| **해결** | 구조 분해 = `className` 분리 + `props`에 나머지 흡수로 이해. `className` 안 주면 `undefined`, `ref`는 `forwardRef`로 별도 인자 |

### 3. ODsay 추천 경로가 “하나뿐”처럼 보임 / UX

| | |
|--|--|
| **문제** | 네이버지도는 강남→인천공항 등에서 경로가 여러 개인데, 여유로는 최단/쾌적 중심이라 **카드가 1장**이거나 다양성이 없음. 일부 데모 OD는 카드가 시청↔동대문 외엔 1장만 나옴. 시청·동대문 검색이 안 뜨는 경우도 |
| **원인** | (1) UI/어댑터가 다중 `path`를 N장 카드로 안 풀었거나 샘플 OD가 단일 경로. (2) 로컬 역 검색 사전·정규화 누락. (3) 쾌적/혼잡까지 카드에 넣으면 설계가 커짐 |
| **해결** | **1차:** 다중 경로 → `RouteOptionCard` N장 + `RouteTimelineBar`(시간·호선색·환승·도보). 픽스처·`demo-route-pairs` 보강. 타임라인 시각(두께·pill·호선 원/도보 시간) 조정. **보류:** 경로별 쾌적·혼잡 표시는 시안 후 진행. **미완:** 해당 UI는 로컬만, PR 없음 |

### 6. 노선도에서 호선이 거의 안 보임 (핵심 버그) — [PR #1](https://github.com/davidmh0203/SUBWAY-PREDICT/pull/1)

| | |
|--|--|
| **문제** | 팀원 쪽에서 노선도 페이지에 **한 호선(체감상 거의 한 줄)만** 보임. “main/backend/integration이 안 맞아서?”로 의심 |
| **원인** | 브랜치 불일치가 **직접 원인 아님**. 기술적 원인: `seoulOnly`(키가 `1호선`~`8호선`일 때만 통과) + 세그먼트 HEX가 **공식 상징색**으로 저장됨 + 호선 키 조회는 **SVG HEX 맵** 위주 → 대부분 탈락. `#CD7C2F`(6호선)만 SVG≈공식색이라 **6호선 16세그만** 남음. (`backend` 고립은 협업 혼란만 가중) |
| **해결** | `HEX_TO_LINE_KEY`에 공식색+SVG HEX 통합, `metro-network`가 동일 맵 사용. 회귀 `npm run verify:seoul-lines`(버그 모드 6호선만 / 수정 후 1~8 전부). **PR #1** 오픈, CI success. 팀원 `backend`에도 유사 `lineKeyForSvgHex` 방향 커밋. **미완:** PR `main` 미머지 · backend와 정리 통합 필요 |
