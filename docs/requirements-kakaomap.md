## 1. 개요
사용자가 지하철 혼잡도 예측 프로젝트 내에서 독립적인 지도 화면을 조회할 수 있도록 **카카오맵 API를 연동한 전용 지도 페이지**를 구축한다.
현재 화면 상의 별도 진입 버튼은 추가하지 않으며, 브라우저 주소창에 직접 `#map` 해시 경로를 입력해야 진입할 수 있도록 구현한다.

---

## 2. 세부 요구사항

### 2.1 독립된 지도 화면 제공
- **경로 진입**: 브라우저 주소창에 `/#map` 또는 `#map` 입력 시 이동.
- **화면 구조**:
  - **헤더**: '카카오 지도' 타이틀과 함께 이전 화면(홈)으로 이동할 수 있는 **뒤로가기 버튼** 제공.
  - **지도 영역**:
    - 기본 가이드인 `500x400` 크기를 반응형으로 개선하여, 모바일 뷰포트 너비(`w-full`)에 맞추고 화면 높이에 자연스럽게 어우러지도록 설계 (예: 모바일 화면 기준 높이 `450px` 내외).
    - 모던하고 깔끔한 라운드 모서리(`rounded-2xl`) 및 그림자 효과(`shadow-surface`) 적용.
  - **정보 카드**: 지도 하단 혹은 오버레이 형태로 지도 사용법이나 기본 위치 정보(예: 카카오 판교오피스 및 지하철 연계 정보)를 보여주는 요약 정보 패널 추가.

### 2.2 카카오맵 API 연동
- **API 로드**:
  - SPA(Single Page Application) 환경에 적합하도록, 컴포넌트가 마운트될 때 동적으로 카카오맵 스크립트를 로드.
  - `//dapi.kakao.com/v2/maps/sdk.js?appkey=API_KEY&autoload=false` 형식을 활용하여 비동기 로딩을 지원하고, 스크립트 로드 완료 시 `window.kakao.maps.load()` 콜백을 호출하여 지도를 생성.
- **기본 위치 설정**:
  - 지도 중심점 좌표: 가이드 제공 좌표인 위경도 `(33.450701, 126.570667)` (카카오 본사)를 기본값으로 하되, 프로젝트의 성격(수도권 지하철)을 고려해 서울 지하철역 좌표(예: 시청역 `37.5665, 126.9780`)로 전환이 용이하도록 구현.
  - 지도 확대 레벨: `3`.
- **API 키 관리**:
  - `.env` 파일에 `VITE_KAKAO_MAP_API_KEY` 환경 변수를 추가하여 프론트엔드 코드에서 `import.meta.env.VITE_KAKAO_MAP_API_KEY`로 유연하게 접근할 수 있도록 설계.

### 2.3 라우팅 구현
- `src/App.jsx` 내 `VIEWS` 배열에 `"map"`을 등록.
- `view === "map"` 분기를 설정하여 해당 주소 입력 시 `MapScreen.jsx`를 렌더링.
- 다른 화면의 내비게이션 바(BottomNav)에는 지도 버튼을 추가하지 않아 비공개 주소 진입 형태 유지.
